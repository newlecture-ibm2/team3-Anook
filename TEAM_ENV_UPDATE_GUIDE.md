# 🚨 필수 공지: AI 서버 및 멀티모달 채팅 환경 업데이트 가이드

최근 작업 과정에서 AI 서버의 **Gemini SDK 버전을 최신으로 교체**하고 프론트엔드/백엔드 로직에 여러 최적화가 이루어졌습니다. **이 내용을 동기화하지 않으면 AI 서버가 켜지자마자 죽거나, 모든 요청이 프론트(FRONT) 데스크로 넘어가는 버그가 발생**할 수 있습니다. 

팀원 여러분들은 아래 가이드에 따라 로컬 개발 환경을 업데이트해 주시기 바랍니다!

---

## 1. 🛠️ 변경 사항 요약

1. **AI 서버 패키지 전면 교체 (가장 중요)**
   - 기존의 구형 `google.generativeai` 패키지를 제거하고, 구글의 최신 공식 SDK인 `google-genai`로 완전히 이관했습니다. 
   - `ai/requirements.txt`가 업데이트되었으며, 관련 파이썬 코드(`client.py`, `main.py`)들의 문법이 모두 최신 버전으로 수정되었습니다.
2. **이미지 처리 및 라우팅 로직 최적화**
   - 기존에는 AI가 이미지를 직접 해독하도록 하여 응답 지연과 비용 낭비가 발생했습니다.
   - **수정 후**: AI는 고객의 '텍스트'만으로 빠르고 정확하게 부서를 라우팅하며, 사진(Base64)은 백엔드에서 Redis에 캐싱하여 직원의 대시보드(모달)로만 바로 전달되도록 분리 설계했습니다.
3. **음성 메시지 자동 전송 (UX 개선)**
   - 투숙객 채팅창(`ChatInput.tsx`)에서 마이크 버튼을 눌러 음성 인식을 진행한 후, 인식이 종료되면 전송 버튼을 누르지 않아도 **자동으로 메세지가 발송되도록 UX를 개선**했습니다.
4. **백엔드 Fallback 로직 확인**
   - AI 서버가 죽어 있거나 통신 에러가 발생할 경우, 백엔드는 이를 무시하지 않고 티켓을 `FRONT` 부서의 `URGENT(긴급)` 상태로 강제 이관하는 안전장치가 작동 중입니다. (만약 모든 챗봇 질문이 프론트로 간다면 AI 서버가 죽어있는 것입니다!)

## 2. 💻 팀원 필수 실행 명령어 (환경 동기화)

로컬에서 최신 버전을 제대로 구동하기 위해 터미널에서 다음 명령어들을 **반드시 순서대로 실행**해 주세요.

### Step 1. 최신 코드 Pull (필수)
\`\`\`bash
git pull origin dev
\`\`\`

### Step 2. AI 서버 Docker 컨테이너 재빌드 (필수)
새로운 파이썬 패키지(`google-genai`)를 도커 안에 설치해야 하므로, 기존 컨테이너를 내렸다가 **새로 빌드**하여 올려야 합니다. 프로젝트 루트 폴더에서 아래 명령어를 실행하세요.
\`\`\`bash
docker compose -f docker-compose.local.yml down
docker compose -f docker-compose.local.yml up --build -d
\`\`\`

### Step 3. (선택 사항) VSCode 에디터 빨간 줄 해결
로컬 Mac/Windows 환경에 가상환경(venv)이 설정되어 있지 않은 경우, VSCode 등 에디터에서 `import uvicorn`이나 `import fastapi` 부분에 빨간 줄(Linter 에러)이 뜰 수 있습니다. 
실행(Docker)에는 지장이 없지만 꼴 보기 싫다면 본인 컴퓨터 터미널에서 패키지를 한 번 설치해 주시면 됩니다.
\`\`\`bash
cd ai
pip install -r requirements.txt
# (macOS의 경우 'pip3 install -r requirements.txt --break-system-packages' 가 필요할 수 있습니다.)
\`\`\`

## 3. 🔍 확인(테스트) 방법

모든 세팅이 끝난 후, 다음 테스트를 통해 정상적으로 연동되었는지 확인합니다.

1. **로컬 서버 기동**: 프론트(`npm run dev`)와 백엔드(`./gradlew bootRun`)를 각각 실행합니다.
2. **사진 전송 테스트**: `http://localhost:3000/guest/chat` 에 접속 후, **사진 한 장과 "변기가 깨졌어요"** 라는 텍스트를 함께 전송합니다.
3. **분류 확인**: 백엔드 터미널 로그를 확인하여, 해당 티켓이 `FRONT`가 아닌 **`FACILITY(시설)` 부서로 라우팅**되었는지 확인합니다.
4. **음성 전송 테스트**: 마이크 아이콘을 누르고 말을 한 뒤, 멈췄을 때 "전송 버튼"을 누르지 않아도 메시지가 자동 발송되는지 확인합니다.

> 위 테스트가 모두 정상적으로 동작한다면 완벽하게 환경이 동기화된 것입니다! 고생하셨습니다! 👍
