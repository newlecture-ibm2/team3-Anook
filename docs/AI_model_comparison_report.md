# Gemini 2.5 vs 3.5 Flash 모델 비교 테스트 결과

> **결론**: Gemini **2.5 Flash** 유지 결정
> **이유**: 3.5가 비용은 우위지만, 응답 속도가 **2.2배 느려** 게스트 UX 저하

---

## 1. 테스트 배경

- 현재 사용 중인 모델: `gemini-2.5-flash`
- 비교 대상: `gemini-3.5-flash`
- 목적: 더 가성비 좋은 모델 탐색

---

## 2. 테스트 방법

### A/B 테스트 인프라 구축
- `.env` 환경변수로 챌린저 모델 지정 (`GEMINI_AB_MODEL`)
- AI 클라이언트에서 요청마다 두 모델 중 **랜덤 50/50** 분기
- `ai_log` 테이블의 `model_name` 컬럼으로 모델별 자동 집계
- 비교 API 신규 추가: `GET /frontdesk/ai-logs/compare`

### 측정 지표
| 항목 | 의미 |
|---|---|
| `latency_ms` | 응답 속도 |
| `prompt_tokens` + `completion_tokens` | 토큰 사용량 (비용) |
| `is_fallback` | 응답 파싱 실패율 (안정성) |

---

## 3. 테스트 결과

| 항목 | Gemini 2.5 Flash | Gemini 3.5 Flash |
|---|---|---|
| **평균 응답 속도** | **3,916.5ms** ✅ | 8,704.6ms |
| **토큰 사용량** | 많음 | **적음** ✅ |
| **속도 차이** | 기준 | **2.2배 느림** |

### 핵심 발견
- **속도**: 2.5가 압도적으로 빠름
- **비용**: 3.5가 토큰을 적게 사용하여 API 비용 우위
- **트레이드오프**: 비용 우위 vs 속도 우위

---

## 4. 시도해본 추가 최적화

### 4-1. 1-Pass 통합 엔진 (Unified Engine)

> **시도 → 채택 안 함**

**아이디어**
- 기존 2회 직렬 호출(라우터 + 도메인 에이전트)을 1회로 통합

**기대**
- Gemini 호출 횟수 절반 → 응답 속도 50% 단축

**실제 결과**
- 오히려 **더 느려짐**
- 원인: 6개 부서 규칙을 하나의 프롬프트로 합치면서 **입력 토큰이 폭증**
- 호출 횟수 감소 < 프롬프트 크기 증가 영향

**결론**: 기존 2-Pass 구조가 더 효율적이므로 원복

---

### 4-2. Gemini 호출 병렬화 검토

> **검토 → 구조적 불가능**

- 라우터 결과(도메인)가 에이전트 호출의 입력 → **데이터 의존성**
- 병렬화하려면 6개 에이전트를 모두 호출하고 5개를 버려야 함 → 비용 6배

**결론**: 직렬 호출 구조는 유지

---

### 4-3. Thinking Budget 동적 조절

> **인프라 적용 완료 / 본격 측정은 추후**

- Gemini 2.5의 `thinking_budget` 기능 활용
- `-1` (동적): 모델이 입력 복잡도에 따라 자체 판단
- `client.py`에 `ThinkingConfig` 명시 적용

**현재 상태**: 동적 모드 적용 완료, 향후 한계값 튜닝 여지 존재

---

## 5. 최종 의사결정

### 선택: **Gemini 2.5 Flash 유지**

| 판단 기준 | 선택 이유 |
|---|---|
| **게스트 UX** | 채팅 응답 4초 vs 8.7초 — 체감 차이 큼 |
| **호텔 서비스 특성** | 실시간 응답이 비용 절감보다 중요 |
| **확장성** | 향후 thinking budget 튜닝으로 추가 최적화 여지 있음 |

### 부가 산출물
- A/B 테스트 인프라 (`GEMINI_AB_MODEL` env, `/compare` API) 구축 완료
- 향후 신규 모델(예: 차세대 Flash) 출시 시 동일 방법으로 즉시 비교 가능

## 6. 정리

```
2.5 vs 3.5 비교 → 2.5 유지 결정 (속도 우선)
1-pass 통합 시도 → 프롬프트 비대화로 역효과, 원복
병렬화 검토 → 데이터 의존성으로 불가
Thinking budget → 동적 모드 적용, 후속 최적화 여지
```
